# 50 Days of RTL

🚀 **My 50-Day Journey to Mastering RTL (Register-Transfer Level) Design**

Welcome to my **50 Days of RTL** repository! This is my personal challenge where I will solve one RTL design problem every day for 50 days. Each day, I will code, simulate, and document a new hardware module using **Verilog**/**SystemVerilog** or **VHDL**.

## 📅 Challenge Overview

- **Duration:** 50 Days
- **Daily Goal:** Solve 1 RTL coding problem
- **Languages:** Verilog / SystemVerilog / VHDL
- **Tools:** Simulators (ModelSim, Vivado, Verilator, etc.)
- **Focus Areas:** Digital logic design, FSM, combinational & sequential circuits, arithmetic units, memory controllers, and more.

## 🧠 Topics Covered (Tentative)

| Day Range  | Topics |
|------------|--------|
| 1–10       | Combinational Logic (MUX, Decoder, Encoder, Comparator, ALU, etc.) |
| 11–20      | Sequential Logic (Flip-flops, Counters, Shift Registers, FSMs) |
| 21–30      | Advanced FSMs, Pipeline, Sync/Async FIFO |
| 31–40      | Arithmetic Units (Multipliers, Dividers, Floating-point) |
| 41–50      | Memory Controllers, Cache, AXI/AHB Interfaces, Project Integration |

## ✨ Why This Challenge?

- To build a **strong foundation** in digital design.
- To improve **coding and debugging** skills in HDLs.
- To create a **portfolio** of RTL designs for career growth.
- To stay **consistent** and disciplined in learning.
