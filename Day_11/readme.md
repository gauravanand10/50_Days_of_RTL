# Day 11 - Structural Modeling

## Objective

Learn the Structural Modeling style in Verilog by building complex digital circuits using smaller reusable modules. Understand the concept of module instantiation and hierarchical design by implementing a Full Adder using two Half Adders.

## Design

### Half Adder

Inputs:

- Input A (`a`)
- Input B (`b`)

Outputs:

- Sum (`sum`)
- Carry (`carry`)

### Full Adder using Two Half Adders

Inputs:

- Input A (`a`)
- Input B (`b`)
- Carry Input (`cin`)

Outputs:

- Sum (`sum`)
- Carry Output (`cout`)

Internal Signals:

- Intermediate Sum (`sum1`)
- Carry from First Half Adder (`carry1`)
- Carry from Second Half Adder (`carry2`)

## Flow

1. Learn Structural Modeling
2. Design a Half Adder Module
3. Verify Half Adder Functionality
4. Instantiate Two Half Adders
5. Connect Intermediate Signals
6. Generate Final Carry Output
7. Create Testbench
8. Simulate the Design
9. Verify the Full Adder Truth Table

## Learnings

- Structural Modeling
- Module Instantiation
- Hierarchical Design
- Reusable Hardware Modules
- Internal Wire Connections
- Half Adder Design
- Full Adder Design
- Modular RTL Development
- Hardware Composition

## Key Observation

During simulation:

- The first Half Adder computes the sum and carry of inputs `a` and `b`.
- The second Half Adder adds the intermediate sum with the carry input (`cin`).
- The final carry output is generated by OR-ing the carry outputs from both Half Adders.
- Structural Modeling makes complex designs easier to build using smaller reusable modules.

## Assignment

Design a **2-bit Ripple Carry Adder** in Verilog using **Full Adder** modules. The carry output of the first Full Adder should be connected to the carry input of the second Full Adder. Write a testbench to verify all possible combinations of two 2-bit numbers and compare the Ripple Carry Adder with a Carry Lookahead Adder (CLA) in terms of carry propagation delay.

## Assignment Design

### 2-Bit Ripple Carry Adder

Inputs:

- 2-Bit Input A (`a[1:0]`)
- 2-Bit Input B (`b[1:0]`)
- Carry Input (`cin`)

Outputs:

- 2-Bit Sum (`sum[1:0]`)
- Carry Output (`cout`)

Internal Signal:

- Intermediate Carry (`carry`)

## Assignment Flow

1. Design a 1-Bit Full Adder Module
2. Instantiate Two Full Adders
3. Connect Carry Output of First Adder to Carry Input of Second Adder
4. Generate Final Carry Output
5. Create Testbench
6. Apply All Input Combinations
7. Simulate the Design
8. Verify Ripple Carry Operation
9. Compare with Carry Lookahead Adder

## Assignment Learnings

- Ripple Carry Adder Architecture
- Structural Design Using Module Instantiation
- Carry Propagation
- Cascaded Full Adders
- Hierarchical RTL Design
- Testbench Development
- Waveform Analysis
- Ripple Carry Adder vs Carry Lookahead Adder

## Key Observation

For every input combination:

- The first Full Adder computes the Least Significant Bit (LSB).
- Its carry output propagates to the second Full Adder.
- The second Full Adder computes the Most Significant Bit (MSB).
- The final carry output is generated after the carry propagates through both stages.
- Ripple Carry Adders are simple but become slower as the number of bits increases due to carry propagation delay.

## Conclusion

Structural Modeling enables designers to build larger digital systems by connecting smaller reusable modules. Implementing a Full Adder using two Half Adders demonstrates hierarchical hardware design, while the 2-bit Ripple Carry Adder illustrates how carry propagation affects performance. This exercise also introduces the motivation for Carry Lookahead Adders, which improve speed by generating carry signals in parallel rather than waiting for sequential propagation.
